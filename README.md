# 🛒 E-Commerce SQL Data Analysis Project

## 📌 Overview
This project simulates a real-world e-commerce system using SQL, covering end-to-end workflow from database design to business insights generation.

---

## 🗂️ Dataset
- 9 normalized relational tables  
- 80K+ records  
- Includes customers, orders, products, categories, payments, shipping, sellers, and inventory  

---

## ⚙️ Key Features
- Designed relational schema with Primary & Foreign Key constraints  
- Loaded data using CSV files (COPY command)  
- Implemented realistic business logic (returns, payments, delivery status)  
- Structured data for efficient querying and analysis  

---

## 📊 Analysis Performed
- Top selling products & revenue analysis  
- Category-wise revenue contribution  
- Customer Lifetime Value (CLV)  
- Monthly sales trends (using CTE & LAG)  
- Return rate & most returned products  
- Seller performance & inactive sellers  
- Inventory stock alerts  

---

## 🛠️ Tools Used
- SQL (PostgreSQL)  
- Python (Faker for data generation)  

---

## 🚀 Key Insights
- Identified high revenue generating products and categories  
- Found that a small group of customers contributes major revenue  
- Detected high return rates in specific categories  
- Highlighted inactive sellers impacting overall sales  
- Identified low inventory products to prevent stock issues  


---

## 📌 Conclusion
This project demonstrates strong SQL skills including data modeling, advanced querying, and real-world business problem solving.
